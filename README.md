# AI Disk Health Monitor

A Python application that uses AI (via Ollama) to evaluate disk health on Ubuntu servers and generate comprehensive diagnostic reports with actionable recommendations.

## Features

- **Comprehensive Disk Monitoring**: Collects SMART data, disk usage, temperature, and I/O statistics
- **AI-Powered Analysis**: Uses Ollama's LLM (gemma3:4b by default) for intelligent health assessment
- **Actionable Reports**: Generates detailed diagnostic reports with specific recommendations
- **CLI Interface**: Easy-to-use command-line tool for on-demand analysis
- **Fallback Mode**: Works even when AI service is unavailable
- **Multiple Output Formats**: Text reports, JSON output, and file saving options

## Requirements

- Ubuntu/Linux system
- Python 3.8+
- Ollama (for AI analysis)
- smartctl (for SMART data collection, optional but recommended)

## Installation

### 1. Install Dependencies

```bash
# Install Python dependencies
pip install -r requirements.txt

# Install smartctl (for comprehensive SMART data)
sudo apt update
sudo apt install smartmontools

# Install Ollama (for AI analysis)
# Follow instructions at: https://ollama.com/download
```

### 2. Download Ollama Model

The application uses `gemma3:4b` by default, which should already be installed on your system. If not:

```bash
ollama pull gemma3:4b
```

## Usage

### Basic Analysis

```bash
python main.py
```

### Verbose Analysis

```bash
python main.py --verbose
```

### Save Report to File

```bash
python main.py --save
```

### JSON Output


### Use Different AI Model




## Command Line Options

| Option | Short | Description |
|--------|-------|-------------|
| `--verbose` | `-v` | Include detailed analysis in report |
| `--save` | `-s` | Save report to file |
| `--json` | `-j` | Output analysis in JSON format |
| `--model` | `-m` | Ollama model to use (default: gemma3:4b) |
| `--check-deps` | | Check if required dependencies are available |
| `--version` | | Show version information |

## What Gets Analyzed

### Disk Health Metrics

- **SMART Data**: Overall health status, individual attribute analysis
- **Temperature**: Current disk temperature and thermal warnings
- **Disk Usage**: Capacity utilization and growth trends
- **I/O Statistics**: Read/write performance and error rates
- **System Information**: Host details and collection metadata

### AI Analysis Components

1. **Overall Health Assessment**: Current status and confidence level
2. **Specific Issues**: Hardware problems, performance concerns, capacity issues
3. **Risk Assessment**: Failure probability and timeframe predictions
4. **Actionable Recommendations**: Immediate actions and preventive measures
5. **Technical Details**: Key metrics and specific concerns

## Fallback Mode

When Ollama AI service is unavailable, the application automatically falls back to:
- Basic rule-based analysis
- Simple health status determination
- Standard recommendations based on collected metrics
- Clear indication that AI analysis was not performed

## Troubleshooting

### Ollama Not Available
```bash
# Check if Ollama is running
ollama list

# Start Ollama service
sudo systemctl start ollama
```

### smartctl Not Found
```bash
# Install smartctl
sudo apt install smartmontools

# Test smartctl
sudo smartctl -H /dev/sda
```

### Permission Issues
Some disk operations may require root privileges:
```bash
sudo python main.py
```

## Security Notes

- The application only reads disk health information
- No disk modifications or writes are performed
- All data collection is read-only and safe
- AI analysis is performed locally via Ollama
